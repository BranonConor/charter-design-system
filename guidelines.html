Guidelines

UNIFY - The Charter Design System

This system should inform all of our design decisions, across many digital platforms and products. We want to make sure our designs speak to our brand, so our digital footprint is consistent. We'll be leaning on reiteration to most effectively execute this - we want to be able to adapt. That's really all there is to it! It may not be rocket science, but there's a method to the madness nontheless, and properly implementing this system will be very beneficial and powerful moving forward.

Before diving in, be sure to read our Purpose & Vision page so you've got a grasp on the big picture.

Important Elements of UNIFY

Figma

UNIFY's design components live in Figma, a popular modern design tool. You'll notice that this web application uses many embedded pieces of the designs, all directly from Figma. This allows for a highly dynamic design experience. Any changes to the system's master components by designers in our Figma boards will automatically trickle down to this web application in real time, so changes there instantly update here.

The Four Levels of Organization

In its completion, UNIFY will maintain four main levels of organization for designs.

  1. Atomic Components: the small, microscopic stuff. The basic building blocks of everything else, that when arranged in different ways, provide us a framework for efficiently creating bigger and more complicated things. Things like color, typography, spacing, and basic button styles, etc.
  2. Molecular Components: when multiple atomic components come together in a stable way, a molecular component is created. Just like in your high school chemistry class. An example might be something like a form input (that uses a certain text color, a certain background, and certain spacing rules - all atomic components).
  3. Substances: >we can create 'substances' by piecing together molecular components. To play off of our previous example, by piecing together several form inputs and fields (which are strategically composed of our atomic components) we can create a complete form for users to communicate with us.
  4. Materials: this is the highest level of organization for our designs. To continue our ongoing example, this may be an entire 'Contact Page' of a website, which uses the contact form, a navigation menu, text blocks, and other substances. These are essentially our web or digital pages.

CodePen

UNIFY will soon support CodePen, a service that allows developers to essentially play around with snippets of code and store them in the cloud. By utilizing CodePen, we can piece together a component library that has real, interactive, live examples of the components we want to build. Real buttons, forms, menus, etc. can be built and then embedded into UNIFY so anyone can see and interact with the real deal.
